Leetcode-每日一题-lc329矩阵中的最长递增路径

七月 26, 2020 发布在 算法与数据结构, leetcode每日一题

题目描述(leetcode-329)

难度：困难

给定一个整数矩阵，找出最长递增路径的长度。

对于每个单元格，你可以往上，下，左，右四个方向移动。 你不能在对角线方向上移动或移动到边界外（即不允许环绕）。

示例1:

输入: nums = 
[
  [9,9,4],
  [6,6,8],
  [2,1,1]
] 
输出: 4 
解释: 最长递增路径为 [1, 2, 6, 9]。

示例2:

输入: nums = 
[
  [3,4,5],
  [3,2,6],
  [2,2,1]
] 
输出: 4 
解释: 最长递增路径是 [3, 4, 5, 6]。注意不允许在对角线方向上移动。

思路分析:

  这是一个求路径的题目，且每一个状态只和上下左右有关，不难想到记忆化存储+搜索算法。利用动态规划的思路，这道题的状态转移方程也很容易获得。

  不妨考虑对这个算法的优化，朴素来说，可以从每个点开始进行BFS/DFS，最后求出极值；换个想法，也可以考虑从初值开始递推，也就是找到动态规划的起始点。顺着这个思路，将这个矩阵和路线，看成一张有向无环图，只有小的数字能指向大的数字，也就是所有大的数字出度均为0。利用拓扑排序+BFS的想法，建立一个队列存储所有出度为0的点，对这些点进行BFS，并在每一步迭代中将这些点从图中删去（也就是将指向这些点的点的出度减一）。那么最后需要求的最长路径，也就是迭代的步数。

  • 在实际写代码的时候，利用了一个C++新特性，也就是lambda表达式

    auto valid = [&](int x, int y){return x >= 0 && x < matrix.size() && y >= 0 && y < matrix[0].size();};

    可以便捷的定义函数，注意返回值只能是auto，不能是bool，[&]表示以引用方式传递

  • 另外一直debug的一个地方在于，在遍历queue的时候，循环条件不能直接用queue.size()

    因为，for循环时，每一次进行条件判断，而queue的大小是动态的，会出现错误

 最终这个算法的空间复杂度为O(mn)，时间复杂度为O(mn)。
